這是个好问题！小布立刻設定了自動同步機制：

同步腳本

小布寫了一個 Python 腳本 portfolio-sync-v2.py，功能包括：

  • 每天早上自動檢查 Google Drive 資料夾

  • 更新 images.json 並推送 GitHub
  • 發現新照片時主動通知雅布大人

Cron Job 設定

名稱：Portfolio Google Drive Sync
執行時間：每天 08:00（Asia/Taipei）
工作方式：自動檢查 → 對比 → 更新 → 推送

同步流程圖

   ↓

Google Drive JSON 格式注意

gog drive ls 的輸出格式是 {"files": [...]}，不是直接陣列！

解析時要小心：

raw_data = json.loads(result.stdout)
if isinstance(raw_data, dict) and 'files' in raw_data:
    gog_data = raw_data['files']

🐛 遇到的大坑

1. Firebase Config 重複宣告問題

錯誤：Identifier 'firebaseConfig' has already been declared

原因：firebase-config.js 和 script.js 都宣告了 firebaseConfig

解決：移除 script.js 中的 let firebaseConfig = null; 宣告

2. Google Drive API 403 錯誤

錯誤：Requests from referer <empty> are blocked

原因：Google Cloud Console 的 API Key 需要設定「HTTP 參照位址」允許網站

解決：在 Google Cloud Console 允許 https://yangbubu1978-web.github.io

3. 瀏覽器快取導致更新無效

問題：更新程式碼後，瀏覽器還是載入舊的快取版本

解決：

  • 在 script.js URL 加上版本參數：?v=20260403b
  • 使用無痕模式測試
  • 按 Ctrl/Cmd + Shift + R 強制刷新

4. Google Drive API 只回傳 100 張

問題：API 預設只回傳 100 張照片

解決：在 API URL 加入 &pageSize=1000 參數

重要：同時要在 fields 加入 nextPageToken 才能正確處理分頁

// 新增追蹤目前位置
let currentLightboxIndex = 0;
let currentLightboxImages = [];

// Lightbox 開啟時，儲存目前的照片列表
function openLightbox(clickedImage) {
    // ... 原有的開啟邏輯 ...
    
    // 找到目前點的是第幾張
    currentLightboxIndex = images.indexOf(clickedImage);
    currentLightboxImages = images;
    
    // 圖片點擊 → 跳下一張
    lightboxImg.onclick = showNextImage;

// 幻燈片功能
function showNextImage() {
    currentLightboxIndex = (currentLightboxIndex + 1) % currentLightboxImages.length;
    lightboxImg.src = currentLightboxImages[currentLightboxIndex];
    updateSlideIndicator();
}

// 更新指示器
function updateSlideIndicator() {
    slideIndicator.textContent = 
        `${currentLightboxIndex + 1} / ${currentLightboxImages.length}`;
}

✨ 凌晨更新：照片同步、子資料夾支援、API 大維修（2026-04-05 凌晨）

🎯 問題描述

雅布大人發現一個問題：網站只顯示 1179 張照片，但 Google Drive 裡實際有 2611 張！而且子資料夾的照片也看不到。

🔍 調查過程

小布打開瀏覽器 Console 發現一堆錯誤：

第一關：400 Bad Request（pageSize 超出上限）

錯誤：Invalid value '1200'. Values must be within the range: [value: 1, value: 1000]

原因：Google Drive API 的 pageSize 上限是 1000，但我們設了 1200
修復：改成 pageSize: 1000 ✅

第二關：403 Forbidden（Shared Drive 需要特殊參數）

錯誤：The supportsAllDrives parameter was not set to true.

原因：Flickr_Photos 是「共享雲端硬碟」（Shared Drive），需要特別的 API 參數
修復：加入 supportsAllDrives: true 和 includeItemsFromAllDrives: true ✅

第三關：瀏覽器快取（一直載入舊 JavaScript）

問題：HTML 裡的 script.js?v=20260406b 讓瀏覽器一直用快取

修復：把 index.html 裡的版本號改成 ?v=20260405c，強迫瀏覽器載入新檔案 ✅

🐛 學到的大坑

  1. pageSize 有上限：Google Drive API 的 pageSize 最大只能設 1000，不是愈多愈好

💡 重要參數筆記（未來必讀）

fetch() 參數設定：

- pageSize: 1000（最大 1000）

- fields: 'files(id,name,mimeType),nextPageToken'（只要這些欄位，省流量）
- key: GOOGLE_DRIVE_API_KEY
